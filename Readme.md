MineBTC is a Solana-native game with two loops running on the same bet:

- **Round loop:** 60-second arena rounds where players place `country + direction` positions and compete for instant SOL + dBTC rewards.
- **Epoch loop:** the same bets accumulate into an active index market where oracle-updated country rankings determine direction winners and epoch reward distribution.

The short version: **the game is the data pipeline**. Round activity keeps the experience fast and fun. Epoch settlement turns that activity into structured geopolitical conviction.

MineBTC produces **Geopolitical Risk Intelligence**: real-time, crowd-sourced, oracle-settled country signal generated by live gameplay.

## What Is Live In The Contracts

### 1. Country + Direction Betting

Players no longer bet on block numbers.

Every round bet is:

- a **country / faction**
- a **direction**: `Down`, `Neutral`, or `Up`

Users can place:

- a single country-direction bet with `join_round`
- multiple country-direction bets in one transaction with `join_round_batch`

That same bet powers both the round game and the active epoch market.

### 2. Round Resolution

Each round lasts 60 seconds.

When the round closes:

1. the contract selects a random winning **country** from countries that actually received bets
2. it selects a random winning **direction** from active directions on that winning country
3. rewards are distributed as follows:

- exact `country + direction` winners share the main SOL + dBTC round payout
- bettors on the winning country but the wrong direction share the consolation dBTC pool
- stakers on the winning country earn the configured staking emission share
- motherlode can add a jackpot bonus to exact winners

### 3. Epoch Prediction Markets

MineBTC also runs an index-driven epoch layer.

Each epoch has:

- an **active index**
- a **question hash**
- a configurable **duration**
- oracle-updated country scores and rankings

Typical active index families can include themes such as `Economics`, `Military`, `AI Race`, or `Space Race`. The contract treats these as explicit on-chain index states, not hardcoded round modes.

During the epoch, every round bet is accumulated into `country + direction` epoch totals.

When the epoch settles:

- each country resolves to `Down`, `Neutral`, or `Up` based on rank change
- each country receives a rank-weighted reward pool
- only users who chose the correct direction for that country share that country's epoch rewards

The current setup scripts still default to a **24h epoch duration** in [setup_scripts/config.json](/Users/sunshinerider/Desktop/LifeOrDream%20Labs/DogeWifBtc/MineBtc-fi/setup_scripts/config.json), but the contract supports configurable epoch timing and rotating active indexes.

### 4. Doges

Doges have two distinct jobs:

- **Operator doge:** one gameplay doge can be locked to boost weighted round points and carry DNA, XP, and mutation state through live play
- **Staked doges:** separate passive staking multipliers that boost long-term hashpower

Today, round play updates the operator dog's cached DNA / XP / multiplier state. Epoch settlement already uses the same betting flow, which gives the product a clear path toward making long-term doge evolution depend more on epoch accuracy over time.

### 5. Autominers

Autominers now use the same betting surface as manual players.

Supported modes:

- `Specific`: a configured list of country-direction picks
- `Random`: a random set of countries with one shared direction

Old block-number autominer configs are no longer part of the contract API.

## Product Framing

MineBTC should be described as:

- a **live country arena game**
- an **oracle-settled index market**
- a **data pipeline for geopolitical indexes**

MineBTC should not be described as:

- a 24-block betting game
- block-high / block-low betting
- a UI where the player is choosing block numbers

Use this language instead:

- `country`
- `direction`
- `round`
- `epoch`
- `active index`
- `operator doge`

Lead with what is live and verifiable today: the round arena, the shared bet surface, the epoch accounting path, and the on-chain doge/operator system.

```text
    ├── game.rs         # Round start/end and staker payout flow

    ├── stake.rs        # MineBTC / LP staking

    ├── economy.rs      # Price snapshots, emissions, POL
    ├── tax.rs          # Transfer tax distribution
    └── helper.rs       # Shared math and vault helpers
```

## Build And Verify

```bash

anchor build -p minebtc

# Format all Rust files
cargo fmt --all

# Check the MineBTC program
cargo check -p minebtc

# Run the MineBTC unit suite
cargo test -p minebtc --lib
```
