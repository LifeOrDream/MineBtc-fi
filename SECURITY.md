# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the MineBTC program, please report it responsibly.

**Email:** gm@minebtc.fun

**Do NOT open a public GitHub issue for security vulnerabilities.**

Please include:
- Description of the vulnerability
- Steps to reproduce
- Potential impact assessment
- Suggested fix (if any)

## Response Timeline

| Stage | Timeline |
|-------|----------|
| Acknowledgment | 48 hours |
| Initial assessment | 7 days |
| Fix deployed | Depends on severity |

## Scope

**In scope:**
- MineBTC Solana program (`Hw9uxvtmQdS57N6aNwJA5iqjSqzhRDdopCHgm8EPwkqx`)
- Program-derived accounts (PDAs) and associated vaults
- Token-2022 transfer fee logic
- Staking, country-direction betting, epoch settlement, and reward distribution logic
- Autominer execution and gameplay-doge custody / synchronization flows
- NFT minting and bonding curve logic

**Out of scope:**
- Frontend / UI issues
- Third-party dependencies (Raydium, Metaplex Core)
- Social engineering attacks
- Denial-of-service attacks on RPC nodes

## Responsible Disclosure

We ask that you:
- Allow reasonable time for us to address the issue before public disclosure
- Do not exploit the vulnerability beyond what is necessary to demonstrate it
- Do not access or modify other users' data or funds
